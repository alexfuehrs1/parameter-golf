# Skill: TDD Red-Green-Refactor

## Purpose
Enforce strict TDD workflow:
Red → Green → Refactor

---

## Instructions

For any implementation task:

1. RED
   - Write or modify a test that captures the desired behavior
   - Ensure it FAILS
   - Explain why it fails

2. GREEN
   - Implement the MINIMAL code needed
   - Do not over-engineer
   - Do not refactor yet

3. VERIFY
   - Ensure the test would pass
   - Ensure no unrelated behavior is affected

4. REFACTOR (optional)
   - Only if tests are green
   - No behavior changes allowed

---

## Rules

- Never skip the RED phase
- Never combine multiple behavior changes
- Keep iterations extremely small
- Always explain reasoning
