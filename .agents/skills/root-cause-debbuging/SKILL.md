# Skill: Root Cause Debugging

## Purpose
Fix underlying causes, not symptoms.

---

## Process

1. Reproduce the issue
2. Form hypotheses
3. Narrow down the cause
4. Add instrumentation if needed
5. Confirm root cause
6. Write failing regression test
7. Apply minimal fix

---

## Rules

- No blind patching
- No guessing
- Always validate assumptions

---

## Output

- Observed behavior
- Hypothesis
- Root cause
- Fix
- Regression test
