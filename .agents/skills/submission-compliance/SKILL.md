# Skill: Submission Compliance

## Purpose
Reduce disqualification risk and ensure the result is submit-ready.

## Instructions

Before calling a result submission-worthy, verify that the package includes:
- model weights
- config
- supporting code
- training log
- execution script
- short write-up

Also verify:
- artifact-size compliance
- runtime-budget compliance
- no unresolved legality / leakage concern
- no missing reproduction detail

## Rules

- Do not call something “ready to submit” unless compliance was checked explicitly
- If legality is uncertain, label it as uncertain
- Conservative interpretation beats clever ambiguity
